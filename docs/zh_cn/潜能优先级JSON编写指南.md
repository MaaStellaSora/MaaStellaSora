## 1. 什么是潜能优先级 JSON？

在爬塔时，每一层都会让你从三个潜能中选择一个。这个 JSON 配置文件的作用，就是**告诉程序你希望优先选择哪些潜能、在什么条件下选择**。

程序会按照你写的顺序，从上到下依次检查每一条规则，找到当前可选潜能中**排名最高**的那一个，自动点击它。

如果所有规则都不匹配，程序会自动选择系统推荐的潜能作为兜底。

---

## 2. JSON 基础知识速览

如果你完全不了解 JSON，先看这里；已经熟悉的可以跳过。

JSON 是一种用文字表示数据的格式。本配置中你只需要了解两种结构：

**对象（Object）**：用花括号 `{}` 包裹，里面是 `"键": 值` 的组合。
```json
{
    "trekker": "夏花",
    "potential": "螺旋风涡"
}
```

**列表（Array）**：用方括号 `[]` 包裹，里面是多个元素，元素之间用逗号分隔。
```json
["螺旋风涡", "风华永续", "劲风利矢"]
```

本配置的整体结构是一个**列表**，里面放多个**对象**，每个对象就是一条潜能规则。

## 3. 最简单的配置：只写潜能名

最基础的规则只需要一个 `potential` 字段，表示"我想要这个潜能"。

```json
[
    {"potential": "螺旋风涡"}
]
```

这条规则的含义：**只要当前可选中有"螺旋风涡"，就选它。**

多条规则时，**越靠前的排名越高**：

```json
[
    {"potential": "螺旋风涡"},
    {"potential": "风华永续"},
    {"potential": "飞花乱坠"}
]
```

含义：优先选螺旋风涡；没有则选风华永续；还没有则选飞花乱坠。三者都没有时，程序选择系统推荐。

---

## 4. 标注潜能归属角色：trekker

`trekker` 字段用于标注这个潜能属于哪个角色。**强烈建议填写**，程序会用它来统计你已拥有某个角色的潜能数量，支持后续的数量条件判断（见第10节）。

```json
[
    {"trekker": "夏花", "potential": "螺旋风涡"},
    {"trekker": "花原", "potential": "飞花乱坠"}
]
```

不填写 `trekker` 时，程序仍然正常运行，但该潜能被选中后会归入 `unknown` 分组，并在日志中输出 debug 信息。

---

## 5. 添加等级跨度限制：level_span

`level_span` 表示**等级跨度**，即这次升级能升多少级（`新等级 - 旧等级`）。

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `level_span` | 整数 | `1` | 只有当跨度 ≥ 该值时，这条规则才生效 |

### 示例：只在能升 2 级以上时才选螺旋风涡

```json
[
    {"trekker": "夏花", "potential": "螺旋风涡", "level_span": 2},
    {"trekker": "夏花", "potential": "风华永续"}
]
```

含义：
- 如果螺旋风涡这次能升 **2 级或以上**，优先选它。
- 如果只能升 1 级，跳过这条规则，改选风华永续。
- 如果风华永续也不在选项里，程序选择系统推荐。

---

## 6. 添加等级上限限制：max_level

`max_level` 表示**该潜能当前等级的上限**，达到或超过这个等级后就不再选它。

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `max_level` | 整数 | `6` | 只有当该潜能**当前等级 < max_level** 时，这条规则才生效 |

### 示例：飞花乱坠升到 1 级后就不再选了（只拿新种类）

```json
[
    {"trekker": "花原", "potential": "飞花乱坠", "max_level": 1}
]
```

含义：飞花乱坠**当前等级为 0**（从未选过）时才选它，选过一次后不再选。

> **什么时候用？** 867 打法中需要铺满花原的潜能种类但不升级，此时对每个花原潜能设置 `max_level: 1` 即可。

---

## 7. 添加刷新次数限制：refresh

`refresh` 表示**本次潜能选择已刷新次数的下限**，只有刷新次数达到该值后，这条规则才会生效。

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `refresh` | 整数 | `0` | 只有当已刷新次数 ≥ 该值时，这条规则才生效 |

默认值为 0，表示不限制，任何时候都可生效。

### 示例：刷新 2 次后才启用兜底规则

```json
[
    {"trekker": "夏花", "potential": "螺旋风涡", "level_span": 2},
    {"trekker": "夏花", "potential": "螺旋风涡", "refresh": 2}
]
```

含义：
- 第 1 条：螺旋风涡跨度 ≥ 2 时，高排名选它（任何时候都生效）。
- 第 2 条：已刷新 **2 次或以上**后，无论跨度多少都选螺旋风涡（兜底）。

> **什么时候用？** 当你希望前几次刷新坚持高标准，刷了一定次数后再降低要求选一个次优选项，避免一直刷不到满意结果时选择系统推荐。

---

## 8. 同时配置多个属性

`level_span` 和 `max_level` 可以同时使用，两个条件都需要满足，规则才生效。

### 示例：螺旋风涡未满 5 级，且这次能升 2 级，才优先选

```json
[
    {"trekker": "夏花", "potential": "螺旋风涡", "level_span": 2, "max_level": 5},
    {"trekker": "夏花", "potential": "螺旋风涡"},
    {"trekker": "夏花", "potential": "风华永续"}
]
```

这里写了**两条螺旋风涡规则**：
- 第一条：螺旋风涡 < 5 级 **且** 跨度 ≥ 2 时，高排名选它。
- 第二条：螺旋风涡满足基础要求时，低排名兜底选它。
- 第三条：前两条都不满足时选风华永续。

---

## 9. 一条规则匹配多个潜能名：potential 为列表

`potential` 除了填一个字符串，也可以填**多个名字的列表**。

```json
[
    {"trekker": "夏花", "potential": ["风华永续", "劲风利矢"], "level_span": 2}
]
```

这条规则的含义：**风华永续和劲风利矢视为同等排名**，当两者都在选项里时：
- 优先选**升级跨度更大**的那个。
- 如果跨度相同，则**按候选列表顺序取第一个**。

> **什么时候用？** 当你觉得几个潜能"差不多，都行"时，可以把它们归为同一条规则。

---

## 10. 添加等级生效条件：condition 等级判断

`condition` 字段可以让某条规则**只在你已经拥有特定潜能达到一定等级时才生效**。

### 基本结构

```json
{
    "trekker": "夏花",
    "potential": "螺旋风涡",
    "condition": [
        {"potential": "风华永续", "level_at_least": 3}
    ]
}
```

condition 列表中的每一项都是一个条件，等级判断包含：
- `potential`：已拥有的潜能名称
- `level_at_least`：该潜能当前等级 **≥** 该值时条件成立（含）
- `level_at_most`：该潜能当前等级 **≤** 该值时条件成立（含，可与 `level_at_least` 同时使用表达区间）

**列表中多个条件默认是 AND 关系，即全部满足才生效。**

### 示例：只有风华永续 ≥ 3 级时，才优先选螺旋风涡

```json
[
    {
        "trekker": "夏花",
        "potential": "螺旋风涡",
        "condition": [
            {"potential": "风华永续", "level_at_least": 3}
        ]
    },
    {"trekker": "夏花", "potential": "风华永续"}
]
```

含义：
- 如果我**已经有 3 级或以上的风华永续**，优先选螺旋风涡。
- 否则这条规则无效，回退到第二条：先把风华永续升上去。

### 示例：需要同时满足两个条件（AND）

```json
[
    {
        "trekker": "夏花",
        "potential": "螺旋风涡",
        "condition": [
            {"potential": "风华永续", "level_at_least": 3},
            {"potential": "劲风利矢", "level_at_least": 2}
        ]
    }
]
```

含义：只有**风华永续 ≥ 3 级 且 劲风利矢 ≥ 2 级**，才优先选螺旋风涡。

## 11. 添加数量生效条件：condition 数量判断

除了判断等级，condition 还支持判断**某个角色已拥有的潜能种类数**。

| 字段 | 说明 |
|------|------|
| `trekker` | 要统计的角色名 |
| `count_at_least` | 该角色已拥有潜能种数 **≥** 该值时条件成立（含） |
| `count_at_most` | 该角色已拥有潜能种数 **≤** 该值时条件成立（含） |
| `level_at_least` | 统计范围限定：只统计当前等级 **≥** 该值的潜能（含，可选） |
| `level_at_most` | 统计范围限定：只统计当前等级 **≤** 该值的潜能（含，可选） |

`level_at_least` 和 `level_at_most` 为可选字段，不填时统计该角色所有已拥有潜能。填写时会先按等级筛选再统计种数。

> **注意**：归入 `unknown` 分组的潜能不参与此统计。建议为所有规则填写 `trekker` 字段以确保统计准确。

### 示例：花原凑满 8 种后才开始升夏花关键潜能（867打法核心）

```json
[
    {
        "trekker": "夏花",
        "potential": "螺旋风涡",
        "condition": [
            {"trekker": "花原", "count_at_least": 8}
        ]
    },
    {
        "trekker": "花原",
        "potential": "飞花乱坠",
        "max_level": 1
    }
]
```

含义：
- 花原已拥有 **8 种**潜能后，优先选夏花的螺旋风涡。
- 否则先选花原的飞花乱坠（且只拿一次，`max_level: 1`）。

### 示例：花原拥有至少 5 种等级 ≥ 3 的潜能时才触发

```json
[
    {
        "trekker": "夏花",
        "potential": "螺旋风涡",
        "condition": [
            {"trekker": "花原", "count_at_least": 5, "level_at_least": 3}
        ]
    }
]
```

含义：花原已拥有的潜能中，等级 **≥ 3** 的种数达到 **5 种或以上**时，才优先选螺旋风涡。

## 12. 多条件任意满足：condition OR 逻辑

当你希望"满足其中一个条件就行"时，把条件用**内层列表**包起来，形成 OR 逻辑。

### 结构对比

```json
// AND 逻辑（默认）：condition 里直接放 dict
"condition": [
    {"potential": "风华永续", "level_at_least": 3},
    {"potential": "劲风利矢", "level_at_least": 2}
]
// 含义：风华永续 ≥ 3 AND 劲风利矢 ≥ 2

// OR 逻辑：condition 里放内层 list
"condition": [
    [{"potential": "风华永续", "level_at_least": 3}],
    [{"trekker": "花原", "count_at_least": 8}]
]
// 含义：风华永续 ≥ 3 OR 花原已有 8 种
```

### 示例：攻击流或数量流都能触发

```json
[
    {
        "trekker": "夏花",
        "potential": "螺旋风涡",
        "condition": [
            [{"potential": "风华永续", "level_at_least": 3}, {"potential": "劲风利矢", "level_at_least": 2}],
            [{"trekker": "花原", "count_at_least": 8}]
        ]
    }
]
```

含义：
- 如果**风华永续 ≥ 3 级 且 劲风利矢 ≥ 2 级**，优先选螺旋风涡。
- **或者**花原已拥有 **8 种**潜能，也优先选螺旋风涡。

## 13. 排名规则：列表顺序就是排名顺序

整个配置是一个列表，**越靠前，排名数字越小，优先级越高**。程序在当前三个可选潜能中，会选出排名最小（最优先）的那一个。

日志中会输出每个潜能的排名数字，例如：

```
[潜能识别] 螺旋风涡 | 等级 2→3 | 排名 1
[潜能识别] 风华永续 | 等级 1→2 | 排名 3
[潜能识别] 飞花乱坠 | 等级 0→1 | 排名 无
[潜能选择] 螺旋风涡 | 排名 1
```

**排名数字直接对应你 JSON 列表的行号**，看到日志后可以直接定位到对应规则进行排查。

## 14. 综合示例：867打法

目标：先铺满花原 8 种潜能（每种只拿一次不升级），再优先升夏花关键潜能。

```json
[
    {
        "trekker": "夏花",
        "potential": "螺旋风涡",
        "condition": [
            {"trekker": "花原", "count_at_least": 8}
        ]
    },
    {
        "trekker": "夏花",
        "potential": ["风华永续", "劲风利矢"],
        "condition": [
            {"trekker": "花原", "count_at_least": 8}
        ]
    },
    {
        "trekker": "夏花",
        "potential": "花影旋风",
        "condition": [
            {"trekker": "花原", "count_at_least": 8}
        ]
    },
    {"trekker": "花原", "potential": "飞花乱坠",  "max_level": 1},
    {"trekker": "花原", "potential": "幻影随行",  "max_level": 1},
    {"trekker": "花原", "potential": "疾风怒涛",  "max_level": 1},
    {"trekker": "花原", "potential": "遥远祈福",  "max_level": 1},
    {"trekker": "花原", "potential": "绚烂涡流",  "max_level": 1},
    {"trekker": "花原", "potential": "凌空穿云",  "max_level": 1},
    {"trekker": "花原", "potential": "飒沓如星",  "max_level": 1},
    {"trekker": "花原", "potential": "繁花绽放",  "max_level": 1}
]
```

### 逐条解读

| 条目 | 含义 |
|------|------|
| 第 1 条 | 花原凑满 8 种后，最优先升夏花螺旋风涡 |
| 第 2 条 | 花原凑满 8 种后，升夏花风华永续或劲风利矢（跨度大者优先） |
| 第 3 条 | 花原凑满 8 种后，升夏花花影旋风 |
| 第 4~11 条 | 花原各潜能只拿一次（`max_level: 1`），凑满 8 种 |

## 15. 常见错误与注意事项

### ❌ 忘记写兜底规则

```json
// 不推荐：只写了高要求规则
[
    {"trekker": "夏花", "potential": "螺旋风涡", "level_span": 3}
]
```

如果螺旋风涡跨度不足 3，程序匹配不到任何规则，只能识别系统推荐。建议补一条无条件的兜底：

```json
[
    {"trekker": "夏花", "potential": "螺旋风涡", "level_span": 3},
    {"trekker": "夏花", "potential": "螺旋风涡"}
]
```

---

### ❌ condition 和 potential 搞反

condition 里写的是**你已经拥有的潜能**（已升过的），不是你想选的潜能。

```json
// ❌ 错误理解：以为 condition 是"我想同时选这两个"
{
    "potential": "螺旋风涡",
    "condition": [{"potential": "风华永续", "level_at_least": 1}]
}
// ✅ 正确理解：我已经有 1 级或以上的风华永续，所以这时优先选螺旋风涡
```

---

### ❌ condition 写成 dict 而不是 list

```json
// ❌ 错误
"condition": {"potential": "风华永续", "level_at_least": 2}

// ✅ 正确：condition 必须是列表
"condition": [{"potential": "风华永续", "level_at_least": 2}]
```

---

### ❌ OR 逻辑写法弄混

```json
// ❌ 这是 AND 逻辑（所有条件都要满足）
"condition": [
    {"potential": "风华永续", "level_at_least": 2},
    {"trekker": "花原", "count_at_least": 8}
]

// ✅ 这才是 OR 逻辑（满足其中一个即可）
"condition": [
    [{"potential": "风华永续", "level_at_least": 2}],
    [{"trekker": "花原", "count_at_least": 8}]
]
```

OR 逻辑的标志是：condition 列表里放的是**内层列表**，而不是直接放 dict。

---

### ⚠️ max_level 判断的是当前等级

`max_level: 1` 的意思是：该潜能**当前等级 < 1**（即等级为 0）时生效。

如果当前等级已经是 1，即便你设置 `max_level: 1`，这条规则也不会生效。

---

### ⚠️ 建议为每条规则填写 trekker

不填 `trekker` 时，被选中的潜能会进入 `unknown` 分组，不参与 `count_at_least` / `count_at_most` 统计。
如果你的配置中使用了数量判断（`count_at_least` / `count_at_most`），漏填 `trekker` 会导致统计不准确。

---

## 16. 快速参考：字段一览表

### 规则顶层字段

| 字段 | 类型 | 是否必填 | 默认值 | 说明 |
|------|------|----------|--------|------|
| `trekker` | 字符串 | 建议填写 | — | 潜能归属角色名，用于数量统计 |
| `potential` | 字符串 或 字符串列表 | **必填** | — | 目标潜能名称，填列表时视为同等排名；同优先级且同跨度时取候选列表中的第一个 |
| `level_span` | 整数 | 选填 | `1` | 要求的最小升级跨度（新等级 - 当前等级 ≥ 该值） |
| `max_level` | 整数 | 选填 | `6` | 当前等级必须 < 该值，规则才生效 |
| `refresh` | 整数 | 选填 | `0` | 已刷新次数必须 ≥ 该值，规则才生效；0 表示不限制 |
| `condition` | 列表 | 选填 | `[]`（无条件） | 生效条件；列表中放 dict 为 AND，放 list 为 OR |

### condition 子项字段

**等级判断：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `potential` | 字符串 | 已拥有的潜能名称 |
| `level_at_least` | 整数 | 当前等级 **≥** 该值时条件成立（含） |
| `level_at_most` | 整数 | 当前等级 **≤** 该值时条件成立（含） |

**数量判断：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `trekker` | 字符串 | 要统计的角色名 |
| `count_at_least` | 整数 | 该角色已拥有潜能种数 **≥** 该值时条件成立（含） |
| `count_at_most` | 整数 | 该角色已拥有潜能种数 **≤** 该值时条件成立（含） |
| `level_at_least` | 整数 | 统计范围限定：只统计当前等级 **≥** 该值的潜能（含，可选） |
| `level_at_most` | 整数 | 统计范围限定：只统计当前等级 **≤** 该值的潜能（含，可选） |
