## 1. 什么是潜能优先级 JSON？

在星塔爬塔时，每一层都会让你从三个潜能中选择一个。这个 JSON 配置文件的作用，就是**告诉程序你希望优先选择哪些潜能、在什么条件下选择**。

程序会按照你写的顺序，从上到下依次检查每一条规则，找到当前可选潜能中**符合条件、优先级最高**的那一个，自动点击它。

---

## 2. JSON 基础知识速览

如果你完全不了解 JSON，先看这里；已经熟悉的可以跳过。

JSON 是一种用文字表示数据的格式。本配置中你只需要了解两种结构：

**对象（Object）**：用花括号 `{}` 包裹，里面是 `"键": 值` 的组合。
```json
{
    "name": "攻击力",
    "max_level": 6
}
```

**列表（Array）**：用方括号 `[]` 包裹，里面是多个元素，元素之间用逗号分隔。
```json
["攻击力", "防御力", "生命值"]
```

本配置的整体结构是一个**列表**，里面放多个**对象**，每个对象就是一条潜能规则。

---

## 3. 最简单的配置：只写名字

最基础的规则只需要一个 `name` 字段，表示"我想要这个潜能"。

```json
[
    {"name": "攻击力"}
]
```

这条规则的含义：**只要当前可选中有"攻击力"，就选它。**

---

多条规则时，**越靠前的优先级越高**：

```json
[
    {"name": "攻击力"},
    {"name": "防御力"},
    {"name": "生命值"}
]
```

含义：优先选攻击力；如果没有攻击力，就选防御力；还没有，就选生命值。三者都没有时，程序会识别系统推荐的潜能。

---

## 4. 添加等级跨度限制：level_span

`level_span` 表示**等级跨度**，即这次升级能升多少级（`新等级 - 旧等级`）。

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `level_span` | 整数 | `1` | 只有当跨度 ≥ 该值时，这条规则才生效 |

### 示例：只在能升 2 级以上时才选攻击力

```json
[
    {"name": "攻击力", "level_span": 2},
    {"name": "防御力"}
]
```

含义：
- 如果攻击力这次能升 **2 级或以上**，优先选攻击力。
- 如果攻击力只能升 1 级，跳过这条规则，改选防御力。
- 如果防御力也不在选项里，程序识别系统推荐。

> **什么时候用？** 当某个潜能"少升没意义，多升才值得"时使用。例如有些潜能升 1 级效果微弱，升 2 级才有质变。

---

## 5. 添加最大等级限制：max_level

`max_level` 表示**该潜能当前等级的上限**，超过这个等级后就不再选它。

| 字段 | 类型 | 默认值 | 含义 |
|------|------|--------|------|
| `max_level` | 整数 | `6` | 只有当该潜能**旧等级 < max_level** 时，这条规则才生效 |

### 示例：攻击力升到 4 级后就不再优先选了

```json
[
    {"name": "攻击力", "max_level": 4},
    {"name": "防御力"}
]
```

含义：
- 攻击力**当前等级小于 4** 时，优先选攻击力。
- 攻击力已经是 4 级或更高时，跳过这条规则，改选防御力。

> **什么时候用？** 当某个潜能"前期很强，到一定等级后就不需要继续堆"时使用。

---

## 6. 同时配置多个属性

`level_span` 和 `max_level` 可以同时使用，两个条件都需要满足，规则才生效。

### 示例：攻击力未满 5 级，且这次能升 2 级，才选它

```json
[
    {"name": "攻击力", "level_span": 2, "max_level": 5},
    {"name": "攻击力"},
    {"name": "防御力"}
]
```

注意这里写了**两条攻击力规则**：
- 第一条：攻击力 < 5 级 **且** 跨度 ≥ 2 时，高优先级选它。
- 第二条：攻击力满足基础要求（跨度 ≥ 1，等级 < 6）时，低优先级选它。
- 第三条：兜底选防御力。

这样的写法可以区分"非常想要"和"可以接受"两种情况。

---

## 7. 一条规则匹配多个潜能名：name 为列表

`name` 除了填一个字符串，也可以填**多个名字的列表**。

```json
[
    {"name": ["攻击力", "暴击率"], "level_span": 2}
]
```

这条规则的含义：**攻击力和暴击率视为同等优先级**，当两者都在选项里时：
- 优先选**升级跨度更大**的那个。
- 如果跨度相同，则**随机选**一个。

> **什么时候用？** 当你觉得几个潜能"差不多，都行"时，可以把它们归为同一条规则，避免写多条重复的配置。

---

## 8. 添加生效条件：condition（AND 逻辑）

`condition` 字段可以让某条规则**只在你已经拥有特定潜能时才生效**。

### 基本结构

```json
{
    "name": "某潜能",
    "condition": [
        {"name": "已有潜能A", "level": 2},
        {"name": "已有潜能B", "level": 4}
    ]
}
```

condition 列表中的每一项都是一个条件，每个条件包含：
- `name`：已拥有的潜能名称
- `level`：该潜能的等级要求（当前等级 **≥** 该值时满足）

**列表中多个条件默认是 AND 关系，即全部满足才生效。**

---

### 示例：只有攻击力 ≥ 3 级时，才优先选暴击率

```json
[
    {
        "name": "暴击率",
        "condition": [
            {"name": "攻击力", "level": 3}
        ]
    },
    {"name": "攻击力"}
]
```

含义：
- 如果我**已经有 3 级以上的攻击力**，那么优先选暴击率（用来配合高攻击力）。
- 如果攻击力还没到 3 级，这条规则无效，回退到第二条：先把攻击力升上去。

---

### 示例：需要同时满足两个条件（AND）

```json
[
    {
        "name": "暴击伤害",
        "condition": [
            {"name": "攻击力", "level": 3},
            {"name": "暴击率", "level": 2}
        ]
    },
    {"name": "暴击率"},
    {"name": "攻击力"}
]
```

含义：只有**攻击力 ≥ 3 级 且 暴击率 ≥ 2 级**，才优先选暴击伤害。否则先按顺序补暴击率和攻击力。

---

## 9. 多条件任意满足：condition（OR 逻辑）

当你希望"满足其中一个条件就行"时，把条件用**内层列表**包起来，形成 OR 逻辑。

### 结构对比

```json
// AND 逻辑（默认）：condition 里直接放 dict
"condition": [
    {"name": "A", "level": 2},
    {"name": "B", "level": 3}
]
// 含义：A≥2 AND B≥3

// OR 逻辑：condition 里放 list（内层 list 内部是 AND）
"condition": [
    [{"name": "A", "level": 2}, {"name": "B", "level": 3}],
    [{"name": "C", "level": 6}]
]
// 含义：(A≥2 AND B≥3) OR (C≥6)
```

---

### 示例：攻击流或防御流都能触发

```json
[
    {
        "name": "终极爆发",
        "condition": [
            [{"name": "攻击力", "level": 4}, {"name": "暴击率", "level": 3}],
            [{"name": "防御力", "level": 5}]
        ]
    },
    {"name": "攻击力"},
    {"name": "防御力"}
]
```

含义：
- 如果**攻击力 ≥ 4 级 且 暴击率 ≥ 3 级**（走攻击流），优先选终极爆发。
- **或者**如果**防御力 ≥ 5 级**（走防御流），也优先选终极爆发。
- 否则先补攻击力和防御力。

---

## 10. 优先级排列：列表顺序就是优先顺序

整个配置是一个列表，**越靠前，优先级越高**。程序在当前三个可选潜能中，会选出优先级最高的那一个。

### 示例说明

```json
[
    {"name": "攻击力", "level_span": 3},
    {"name": "暴击率", "level_span": 2},
    {"name": "攻击力"},
    {"name": "暴击率"},
    {"name": "防御力"}
]
```

假设当前三个可选潜能为：
- 攻击力（旧 2 级 → 新 4 级，跨度 2）
- 暴击率（旧 1 级 → 新 3 级，跨度 2）
- 防御力（旧 0 级 → 新 1 级，跨度 1）

程序逐条检查：
1. 攻击力 level_span=3 → 攻击力实际跨度只有 2，**不满足**，跳过。
2. 暴击率 level_span=2 → 跨度 2 **满足**，暴击率得到优先级 4（列表共5条，5-1=4）。
3. 攻击力（无限制）→ 跨度 2 **满足**，攻击力得到优先级 3。
4. 暴击率（无限制）→ 已经匹配过更高优先级，不再更新。
5. 防御力 → 防御力得到优先级 1。

最终选优先级最高的：**暴击率**（优先级 4）。

## 11. 综合复杂示例

下面是一个模拟"完整爬塔策略"的配置，包含了本文档介绍的所有功能：

```json
[
    {
        "name": "攻击力",
        "level_span": 3,
        "max_level": 5
    },
    {
        "name": ["暴击率", "暴击伤害"],
        "level_span": 2,
        "condition": [
            {"name": "攻击力", "level": 3}
        ]
    },
    {
        "name": "终极技能强化",
        "condition": [
            [
                {"name": "攻击力", "level": 4},
                {"name": "暴击率", "level": 3}
            ],
            [
                {"name": "防御力", "level": 5}
            ]
        ]
    },
    {
        "name": "攻击力",
        "max_level": 5
    },
    {
        "name": ["暴击率", "暴击伤害"],
        "condition": [
            {"name": "攻击力", "level": 2}
        ]
    },
    {
        "name": "防御力",
        "max_level": 4
    },
    {
        "name": "生命值"
    }
]
```

### 逐条解读

| 条目 | 含义 |
|------|------|
| 第 1 条 | 攻击力 < 5 级时，如果跨度 ≥ 3 则最高优先选 |
| 第 2 条 | 攻击力已达 3 级后，暴击率/暴击伤害跨度 ≥ 2 时高优先选 |
| 第 3 条 | 已经满足攻击流（攻4+暴击率3）或防御流（防5），则优先拿终极技能 |
| 第 4 条 | 攻击力 < 5 级时普通优先（兜底补攻击力） |
| 第 5 条 | 攻击力 ≥ 2 级后开始补暴击系 |
| 第 6 条 | 防御力 < 4 级时适当选一选 |
| 第 7 条 | 最终兜底：什么都没有就选生命值 |

## 12. 常见错误与注意事项

### ❌ 忘记写兜底规则

```json
// 不推荐：只写了高要求规则，没有低要求兜底
[
    {"name": "攻击力", "level_span": 3}
]
```

如果攻击力跨度不足 3，程序匹配不到任何规则，只能识别系统推荐。建议总是补一条无条件的兜底：

```json
[
    {"name": "攻击力", "level_span": 3},
    {"name": "攻击力"}
]
```

---

### ❌ condition 和 name 搞反

condition 里写的是**你已经拥有的潜能**（已升过的），不是你想选的潜能。

```json
// ❌ 错误理解：以为 condition 是"我想同时选这两个"
{
    "name": "攻击力",
    "condition": [{"name": "暴击率", "level": 1}]
}
// ✅ 正确理解：我已经有 1 级以上的暴击率，所以这时我优先选攻击力
```

---

### ❌ condition 写成 dict 而不是 list

```json
// ❌ 错误
"condition": {"name": "攻击力", "level": 2}

// ✅ 正确：condition 必须是列表
"condition": [{"name": "攻击力", "level": 2}]
```

---

### ❌ OR 逻辑写法弄混

```json
// ❌ 这是 AND 逻辑（所有条件都要满足）
"condition": [
    {"name": "A", "level": 2},
    {"name": "B", "level": 3}
]

// ✅ 这才是 OR 逻辑（满足其中一组即可）
"condition": [
    [{"name": "A", "level": 2}],
    [{"name": "B", "level": 3}]
]
```

OR 逻辑的标志是：condition 列表里放的是**内层列表**，而不是直接放 dict。

---

### ⚠️ max_level 判断的是旧等级

`max_level: 4` 的意思是：该潜能**当前等级（升级前）< 4** 时生效。

如果当前等级已经是 4，即便你设置 `max_level: 4`，这条规则也不会生效。

---

### ⚠️ 同一潜能可以出现在多条规则中

这是有意设计的，用于区分"高优先"和"普通优先"两种情况：

```json
[
    {"name": "攻击力", "level_span": 2},   // 跨度够大时，高优先
    {"name": "攻击力"}                      // 跨度不够时，普通优先兜底
]
```

## 13. 快速参考：字段一览表

| 字段 | 类型 | 是否必填 | 默认值 | 说明 |
|------|------|----------|--------|------|
| `name` | 字符串 或 字符串列表 | **必填** | — | 目标潜能名称，填列表时视为同等优先级 |
| `level_span` | 整数 | 选填 | `1` | 要求的最小升级跨度（新等级 - 旧等级 ≥ 该值） |
| `max_level` | 整数 | 选填 | `6` | 旧等级必须 < 该值，规则才生效 |
| `condition` | 列表 | 选填 | `[]`（无条件） | 基于已拥有潜能的生效条件；列表中放 dict 为 AND，放 list 为 OR |

---

> **提示：** 如果你不确定自己写的配置是否正确，可以把配置直接贴给 AI，描述你的需求，让它帮你检查或生成。这是初学者最有效的上手方式之一。
